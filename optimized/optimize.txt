Optimizations used in the optimized kernels
============================================

Quantize Kernel (quantize_int4_optimized.cu)
--------------------------------------------
1. Warp-level parallelism: 1 warp (32 threads) cooperatively quantizes a 16x64 tile,
   instead of 1 thread per group.

2. Vectorized loads: Each thread loads 8 half values at once via 128-bit loads,
   maximizing memory bandwidth.

3. Warp shuffle reductions: Max absolute value is computed across threads using
   __shfl_xor_sync, avoiding shared memory for the reduction.

4. PTX INT4 packing: Uses the hardware cvt.pack.sat.s4.s32.b32 instruction to
   pack two signed 4-bit values in a single cycle, instead of manual bit shifts.

5. ldmatrix output format: Quantized data is written to shared memory and then
   loaded via ldmatrix.sync.aligned.x4.m8n8.shared.b16, producing a register
   layout directly consumable by tensor core MMA instructions.

6. Scale packing: Scales are packed into a warp-shuffle-friendly layout so the
   GEMM kernel can broadcast them to all lanes with a single __shfl_sync per tile.


GEMM Kernel (gemm_int4_optimized.cu)
-------------------------------------
1. Tensor core MMA: Uses mma.sync.aligned.m16n8k64.row.col.s32.s4.s4.s32, the
   native INT4 x INT4 -> INT32 matrix-multiply-accumulate instruction. Each call
   computes a 16x8 output tile from 64 K elements. Falls back to decomposed
   m8n8k32 on SM75.

2. Large block tiling: BLOCK_M=256, BLOCK_N=128 with 8 warps per block. Each warp
   handles a 32x128 output tile (2x8 micro-tiles of 16x16 each), amortizing
   index/control overhead.

3. Scale broadcasting via warp shuffles: Activation and weight scales are loaded
   once per warp and broadcast to all 32 lanes using __shfl_sync, avoiding
   redundant global memory reads.

4. FP16 fused multiply-add accumulation: INT32 MMA results are converted to FP16
   and accumulated with scale multiplication in a single __hfma2 instruction:
   fpsum += int2half(mma_result) * ascale * wscale.

5. Register-to-global via shared memory transposition: The MMA output register
   layout does not match row-major memory order. The epilogue writes registers
   to shared memory in MMA layout, then reads back in row-major order for
   coalesced global memory stores.

6. Predicated loads: All global memory loads use predication to handle boundary
   conditions without branching, keeping the warp converged.
